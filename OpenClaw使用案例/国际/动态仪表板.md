# 动态仪表板

**分类:** 国际版补充  
**编号:** #07

---

## 🎯 痛点

- 信息孤岛：数据分散在多个平台，切换麻烦
- 实时性差：手动更新报表费时且容易过时
- 视图固定：静态报表无法满足动态决策需求
- 定制困难：现有仪表板工具配置复杂

---

## ⚡ 核心功能

1. **数据聚合** - 从多个数据源（GitHub、Twitter、Google Analytics 等）拉取数据
2. **实时刷新** - 定时更新关键指标
3. **多视图切换** - 支持日/周/月视图，不同维度展示
4. **智能告警** - 指标异常时自动通知
5. **自然语言查询** - 用对话方式获取数据洞察

---

## 🛠️ 所需技能

- `web_fetch` / API 集成 - 数据源接入
- 飞书电子表格 - 数据存储和展示
- 飞书文档 - 嵌入仪表板
- Cron jobs - 数据定时刷新
- `feishu_im_user_message` - 告警推送

---

## 📝 Prompt 模板

```
You are my personal analytics dashboard. Aggregate key metrics:

1. Track these data sources:
   - [GitHub: stars, PRs, issues]
   - [Twitter/X: mentions, followers]
   - [Website: pageviews, unique visitors]
   - [Newsletter: subscribers, open rate]
   - [Custom: your metrics]

2. Present metrics in a daily digest:
   - Today's numbers vs yesterday
   - 7-day trend (up/down arrows)
   - Notable events

3. Answer questions like:
   - "How did last week's engagement compare to the week before?"
   - "What's the trend for the past 30 days?"

4. Alert me when:
   - Any metric changes by >20% day-over-day
   - A milestone is reached (e.g., 1000 followers)

Keep it visual with emojis and arrows. Post to #daily-metrics.
```
