# 活动嘉宾确认

**分类:** 国际版补充  
**编号:** #10

---

## 🎯 痛点

- 确认繁琐：手动发消息、跟进大量嘉宾确认状态
- 状态不清：谁回复了、谁还没回复难以追踪
- 截止紧迫：活动快开始了才发现有嘉宾未确认
- 沟通混乱：多个渠道（邮件、微信、飞书）确认信息不统一

---

## ⚡ 核心功能

1. **嘉宾库管理** - 存储所有嘉宾信息、联系方式、邀请状态
2. **批量邀请** - 一键发送邀请消息到多个渠道
3. **自动跟进** - 未确认的嘉宾自动发送跟进提醒
4. **状态追踪** - 实时更新确认/待定/拒绝状态
5. **统计分析** - 统计回复率、出席率等数据

---

## 🛠️ 所需技能

- 飞书多维表格 - 嘉宾数据库
- `feishu_im_user_message` - 发送邀请和跟进
- `feishu_calendar_event` - 活动关联
- Cron jobs - 定时跟进检查
- 飞书文档 - 邀请模板

---

## 📝 Prompt 模板

```
You are my event guest confirmation manager. Track RSVPs efficiently:

1. Maintain a guest list with:
   - Name, email, phone
   - Event(s) invited to
   - Invitation status: pending/sent → opened → confirmed/declined/maybe
   - Reminder count
   - Dietary restrictions, plus-one info

2. When I share event details:
   - Create guest list entries
   - Send personalized invitations via [飞书/邮件]
   - Set reminder for [X days] before event

3. Every day, check and act:
   - Send follow-up to guests who haven't responded in 5+ days
   - Alert me if <80% confirmed 1 week before event
   - Update status when responses come in

4. Generate event summary:
   - Total invited, confirmed, declined, pending
   - Special requirements (vegan, wheelchair access, etc.)
   - Final headcount

Keep data in [活动嘉宾表] bitable.
```
