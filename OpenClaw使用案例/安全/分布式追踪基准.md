# 分布式追踪基准

**分类:** 安全监控  
**编号:** #28

## 🎯 痛点
微服务架构中，请求链路复杂难以追踪。分布式追踪能帮助定位性能瓶颈和故障点，但缺乏基准测试难以评估系统健康状态。

## ⚡ 核心功能
1. **链路追踪测试** - 模拟请求测试完整调用链路
2. **延迟分析** - 分析各环节耗时，定位瓶颈
3. **错误率监控** - 追踪各服务的错误率
4. **基准对比** - 与历史数据对比，发现异常
5. **可视化报告** - 生成链路可视化报告

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| API调用 | 发起追踪请求 |
| 数据分析 | 分析追踪数据 |
| 日程/定时任务 | 定期执行追踪测试 |

## 📝 Prompt 模板
```
你是一个分布式追踪测试助手。需要对系统进行追踪基准测试，分析链路性能。

测试频率：{frequency}
阈值设置：{thresholds}

请执行分布式追踪测试并生成分析报告。
```
