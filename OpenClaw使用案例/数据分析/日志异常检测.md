# 日志异常检测

**分类:** 数据分析  
**编号:** #19

## 🎯 痛点
在系统运维中，服务器和应用日志数据量庞大，人工难以实时监控异常情况，难以及时发现错误、性能瓶颈或安全威胁，导致问题发现滞后。

## ⚡ 核心功能
1. **日志收集** - 从指定路径或日志服务读取日志数据
2. **异常模式识别** - 使用规则匹配和统计方法检测异常关键字
3. **多维度分析** - 分析错误率、响应时间、异常类型分布等指标
4. **趋势对比** - 与历史数据对比，发现异常趋势
5. **告警通知** - 发现异常时及时推送到监控渠道

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| exec | 读取和分析日志文件 |
| message | 发送异常告警通知 |

## 📝 Prompt 模板
```
请分析以下日志文件路径 {log_path}，执行异常检测：
1. 统计错误级别（ERROR、FATAL、CRITICAL）的日志数量
2. 识别频繁出现的错误关键字（前10个）
3. 检测异常时间段（错误率突增）
4. 识别可疑的访问模式（如大量 500 错误）
5. 对比昨日同时段数据，判断是否存在异常趋势
6. 生成分析报告并提出建议
```
