# GitHub 陈旧 Issue 清理

**分类:** 数据分析  
**编号:** #17

## 🎯 痛点
开源项目中长期存在大量无人响应的陈旧 Issue，占用仓库管理资源，影响社区活跃度，需要定期清理这些过时的问题以保持项目健康。

## ⚡ 核心功能
1. **陈旧 Issue 检测** - 通过 GitHub API 识别超过指定时间无更新的 Issue
2. **自动分类** - 按时间范围（90天/180天/1年）分类陈旧 Issue
3. **智能判断** - 根据标签、里程碑、评论判断 Issue 是否应该保留
4. **批量操作** - 支持添加 stale 标签、发送提醒评论、自动关闭等操作
5. **清理报告** - 生成清理结果报告，包含处理数量和详情

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| github | 搜索陈旧 Issue、执行标签和评论操作 |
| message | 发送清理报告通知 |

## 📝 Prompt 模板
```
请帮我分析 {owner}/{repo} 仓库中超过 90 天无更新的 Issue：
1. 列出所有陈旧 Issue（标题、编号、创建时间、最后更新时间）
2. 按类型分类：已关闭/未解决/需要信息
3. 识别可以安全关闭的 Issue（无标签、无里程碑、无人评论）
4. 生成清理建议报告
5. 执行批量操作：给可关闭的 Issue 添加提醒评论
```

## ⏰ Cron 配置
```json
{
  "schedule": "0 10 * * 0",

}
```
