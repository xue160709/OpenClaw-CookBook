# 邮件自动分类器

**分类:** 数据分析  
**编号:** #21

## 🎯 痛点
日常工作中邮箱会收到大量邮件（客户咨询、内部通知、订阅更新、垃圾邮件等），手动分类耗时且容易遗漏重要邮件，影响工作效率。

## ⚡ 核心功能
1. **邮件抓取** - 通过 IMAP/POP3 或邮件 API 获取邮件
2. **智能分类** - 基于发件人、主题、内容关键词进行自动分类
3. **优先级识别** - 识别紧急/重要的邮件
4. **标签管理** - 自动为邮件添加分类标签
5. **摘要生成** - 为每类邮件生成简要摘要
6. **定时报告** - 定期推送邮件分类汇总

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| web_fetch | 连接邮件服务获取邮件 |
| message | 推送邮件分类摘要 |

## 📝 Prompt 模板
```
请帮我分析最近收到的邮件，分类如下：
- 紧急邮件（客户投诉、系统告警、领导急件）
- 待处理邮件（需要回复的事项）
- 资讯订阅（新闻简报、订阅更新）
- 垃圾邮件（广告、推广）

分类标准：
- 来自重要客户/领导的邮件优先
- 包含 "紧急"、"急"、"重要" 等关键字优先
- 来自已知订阅源的归为资讯类

请返回：
1. 各分类邮件数量统计
2. 紧急邮件清单（发件人、主题、接收时间）
3. 待处理邮件摘要
4. 建议优先处理的邮件
```

## ⏰ Cron 配置
```json
{
  "schedule": "0 * * * *",
  "task": "每小时自动分类新邮件并推送摘要"
}
```
