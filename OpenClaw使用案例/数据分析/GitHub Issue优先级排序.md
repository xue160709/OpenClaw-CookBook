# GitHub Issue 优先级排序

**分类:** 数据分析  
**编号:** #07

## 🎯 痛点
在项目管理中，GitHub Issue 数量繁多且缺乏优先级排序，导致开发团队难以快速识别需要优先处理的问题，影响项目进度和资源分配效率。

## ⚡ 核心功能
1. **Issue 数据抓取** - 通过 GitHub API 获取指定仓库的 Issue 列表
2. **多维度评分** - 基于标签、评论数、创建时间、分配状态等维度计算优先级分数
3. **智能排序** - 将 Issue 按优先级分数降序排列
4. **定期报告** - 定时生成 Issue 优先级报告，推送到指定渠道

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| github | 获取 GitHub Issue 数据、搜索仓库 |
| message | 发送优先级报告到群聊/频道 |

## 📝 Prompt 模板
```
你是一个 GitHub Issue 优先级分析助手。请分析 {owner}/{repo} 仓库中最近创建的 Issue，根据以下维度进行优先级评分：
1. 标签权重（bug > feature > enhancement）
2. 评论数量（越多表示关注度高）
3. 创建时间（越新越优先）
4. 是否已分配（已分配优先）
5. 是否有阻断性问题

请返回按优先级排序的 Issue 列表（前20个），包含 Issue 编号、标题、优先级分数和关键原因。
```

## ⏰ Cron 配置
```json
{
  "schedule": "0 9 * * 1-5",
  "task": "分析项目 Issue 优先级并推送报告"
}
```
