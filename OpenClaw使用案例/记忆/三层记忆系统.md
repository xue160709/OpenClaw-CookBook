# 三层记忆系统

**分类:** 记忆管理  
**编号:** #04

## 🎯 痛点
AI 助手每次会话重启后丢失所有上下文，无法区分长期记忆与短期工作记忆，导致重复询问用户已知信息，或在工作完成后遗忘关键细节。

## ⚡ 核心功能
1. **短期记忆（Working Memory）**：当前会话内的上下文、正在处理的任务、用户偏好
2. **情景记忆（Episodic Memory）**：历史会话中的重要事件、决策、约定
3. **语义记忆（Semantic Memory）**：长期积累的知识、技能、用户画像

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| self-improvement | 记录学习心得、优化记忆策略 |
| obsidian | 管理和查询长期记忆库 |

## 📝 Prompt 模板
```
你是具备三层记忆系统的AI助手。

【短期记忆】当前会话中的任务进度、用户当前需求、正在处理的文件或对话上下文。
【情景记忆】历史上的重要约定、已完成的项目、用户的核心偏好（从 MEMORY.md 和 memory/ 日志中提取）。
【语义记忆】你的核心能力、已知的事实、通用技能。

当用户提出问题时，首先检查：
1. 这个问题是否在短期记忆中有答案？（当前任务相关）
2. 这个问题是否在情景记忆中？（历史交互）
3. 是否有语义记忆可以直接回答？

如果都不确定，再向用户确认。不要重复询问你已经知道的信息。
```

## ⏰ Cron 配置
```json
{

  "task": "memory_tier_sync",
  "description": "每日同步三层记忆，将短期记忆归档到情景记忆"
}
```
