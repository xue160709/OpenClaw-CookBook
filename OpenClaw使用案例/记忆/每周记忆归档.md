# 每周记忆归档

**分类:** 记忆管理  
**编号:** #41

## 🎯 痛点
AI 助手的每日记忆文件（memory/YYYY-MM-DD.md）不断累积，占用大量存储空间且难以检索，同时包含大量无价值的日常流水账，干扰重要信息的获取。

## ⚡ 核心功能
1. **周内总结**：将一周的每日记忆压缩为关键事件列表
2. **归档存储**：将已完成周的日志移动到归档目录
3. **索引生成**：为归档内容生成可搜索的索引标签
4. **重要信息提取**：识别并标记需要长期记住的信息
5. **空间清理**：删除已被整合的原始日志

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| obsidian | 归档管理长期记忆 |
| self-improvement | 记录归档过程中发现的问题 |

## 📝 Prompt 模板
```
你是具备每周记忆归档能力的AI助手。

【归档任务】
每周日执行记忆归档：

1. 读取本周（周一到周日）的所有 memory/YYYY-MM-DD.md 文件
2. 提取关键信息：
   - 用户明确要求记住的事项
   - 重要决策和结论
   - 完成的里程碑
   - 用户的偏好变化
   - 任何需要后续跟进的事项

3. 生成周报格式：
   ## Week of [起始日期]
   - **重要事件**: [列表]
   - **决策**: [列表]
   - **待跟进**: [列表]
   - **用户新偏好**: [列表]

4. 将周报保存到 memory/archive/Week-[年份-周数].md
5. 标记需要长期记忆的信息，更新到 MEMORY.md

【保留原则】
- 完整保留最近4周的每日日志
- 更早的日志归档后可以删除源文件
- 年度总结时整合所有周报

保持归档的轻量化和可追溯性。
```
