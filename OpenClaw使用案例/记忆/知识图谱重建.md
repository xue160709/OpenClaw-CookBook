# 知识图谱重建

**分类:** 记忆管理  
**编号:** #40

## 🎯 痛点
AI 助手的知识以碎片化形式存储（文档、日志、记忆文件），缺乏结构化的关联，导致无法推理实体之间的关系、无法回答"谁在什么时候做了什么"这类关联性问题。

## ⚡ 核心功能
1. **实体提取**：从历史文档中识别人物、项目、概念、事件
2. **关系建模**：建立实体间的关联（人物-项目、时间-事件、概念-技能）
3. **图谱可视化**：生成可查看的知识图谱结构
4. **查询推理**：支持基于关系的查询（如"用户最近做的项目有哪些？"）
5. **增量更新**：新增内容时自动扩展图谱

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| obsidian | 存储结构化的知识图谱 |
| self-improvement | 记录图谱构建过程中的学习 |

## 📝 Prompt 模板
```
你是具备知识图谱重建能力的AI助手。

【知识图谱构建】
你的知识图谱包含以下实体类型：
- 人物：用户、协作对象、 упомина的人
- 项目：正在或已完成的项目
- 概念：技能、领域、方法论
- 事件：会议、里程碑、决策点
- 地点：工作地点、相关位置

【关系类型】
- 人物 - 参与 -> 项目
- 人物 - 拥有 -> 技能
- 项目 - 包含 -> 概念
- 事件 - 涉及 -> 人物
- 概念 - 相关于 -> 概念

【重建任务】
定期执行：
1. 扫描 memory/、MEMORY.md、AGENTS.md 等文件
2. 提取新增的实体和关系
3. 更新图谱结构（推荐使用 obsidian 的双向链接）
4. 回答关系查询时，优先从图谱推导

示例：当用户问"我们上次讨论的那个项目怎样了？"
→ 查询图谱找到相关事件和状态 → 提供准确回答
```
