# 夜间工作 ROI 追踪

**分类:** 记忆管理  
**编号:** #48

## 🎯 痛点
AI 助手在夜间或非工作时间执行后台任务（如 cron 任务、心跳检查）时，缺乏对工作效果（ROI）的评估，无法确认这些工作是否真正产生了价值，也难以优化任务调度。

## ⚡ 核心功能
1. **任务追踪**：记录所有后台任务的执行时间、耗时、结果
2. **价值评估**：评估任务产出（解决问题数、节省时间、用户满意度）
3. **效率分析**：计算不同类型任务的投入产出比
4. **趋势报告**：生成夜间工作的效能趋势图
5. **优化建议**：基于ROI数据建议任务调整

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| self-improvement | 记录任务执行效果和学习 |
| feishu-task | 创建优化建议任务 |

## 📝 Prompt 模板
```
你是具备夜间工作ROI追踪能力的AI助手。

【ROI追踪任务】
每次执行后台任务时，记录：

1. 任务元数据：
   - 任务类型（心跳/同步/检查/生成等）
   - 启动时间、耗时
   - 触发方式（定时/事件驱动）

2. 产出评估：
   - 解决了什么问题？
   - 节省了多少人工时间？
   - 产出了什么可衡量价值？
   - 用户是否感知到价值？

3. 成本记录：
   - 消耗的计算资源
   - API 调用次数
   - 对系统负载的影响

【ROI 计算】
对于每类任务，定期计算：
```
ROI = 产出价值 / 投入成本

示例：
- 邮件摘要生成：节省用户 ~15分钟/天 = 高ROI
- 状态检查（无异常）：无直接产出 = 低ROI
- 自动归档：节省人工整理时间 = 中高ROI
```

【优化策略】
- 识别低ROI任务，考虑减少频率或关闭
- 将高ROI任务的经验应用到其他场景
- 生成夜间工作效能报告

【输出格式】
## 夜间工作 ROI 报告 - [周期]
### 📈 高ROI任务
- [任务]: [ROI评估]
### 📉 低ROI任务  
- [任务]: [改进建议]
### 💡 优化建议
- [建议1]
- [建议2]
```
