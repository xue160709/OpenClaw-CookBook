# 会议纪要自动化

**分类:** 中国特色用例 / 办公自动化  
**难度:** ⭐⭐

---

## 🎯 痛点

- 会议纪要整理耗时
- 会后任务跟进不及时
- 多人会议难以明确责任
- 会议效率难以评估

---

## ⚡ 核心功能

1. **会议接入** - 飞书妙记 / 腾讯会议 / 钉钉会议
2. **语音转写** - 自动生成文字稿
3. **智能摘要** - 提取关键信息、决策、待办
4. **任务创建** - 自动创建飞书/钉钉任务
5. **进度追踪** - 定时提醒待办完成情况

---

## 🛠️ 所需技能

| 技能 | 用途 |
|------|------|
| `feishu_doc` / `dingtalk` | 会议记录获取 |
| `feishu_task` | 任务创建 |
| `memory` | 历史会议索引 |

---

## 📝 Prompt 模板

```
你是会议纪要助手。请根据会议内容：

1. 提取会议基本信息
   - 时间、参与人、主题

2. 总结讨论要点 (3-5 条)

3. 列出决策事项 (如有)

4. 提取待办任务
   - 任务内容
   - 负责人
   - 完成时间

5. 下次会议安排 (如有)

输出格式：Markdown
```

---

## ⚙️ 工作流配置

```yaml
# meeting-pipeline.yaml
workflow:
  - trigger: meeting_end
    source: feishu_miaoji / tencent_meeting
    
  - step: fetch_recording
    action: get_meeting_transcript
    
  - step: generate_summary
    action: ai_summary
    prompt: (见上方模板)
    
  - step: create_tasks
    action: create_feishu_tasks
    assignees: (从纪要提取)
    
  - step: notify
    action: send_to_group
```

---

## ⏰ Cron 配置

```json
{
  "schedule": "0 */2 * * *",
  "task": "check_meeting_tasks",
  "description": "检查待办完成进度"
}
```
