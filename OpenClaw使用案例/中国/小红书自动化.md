# 小红书自动化

**分类:** 中国特色用例 / 社交媒体  
**难度:** ⭐

---

## 🎯 痛点

- 手动发布笔记耗时
- 难以坚持每日更新
- 选题和文案需要大量时间
- 数据分析滞后

---

## ⚡ 核心功能

1. **选题推荐** - 根据账号定位推荐热门选题
2. **文案生成** - 生成符合小红书风格的笔记文案
3. **封面制作** - 生成或推荐封面图
4. **定时发布** - 设置发布时间，自动发布
5. **数据追踪** - 监控笔记曝光、点赞、收藏

---

## 🛠️ 所需技能

| 技能 | 用途 |
|------|------|
| `web_fetch` | 获取热门话题、竞品笔记 |
| `image_gen` | 生成封面图片 |
| `browser` | 自动化发布 |
| `memory` | 选题库和内容库管理 |

---

## 📝 Prompt 模板

```
你是一个小红书内容运营助手。

任务：
1. 每天分析热门话题和爆款笔记
2. 根据我的账号定位推荐 3 个选题
3. 生成符合小红书风格的文案（emoji、段落格式）
4. 建议发布时间

账号定位：[你的领域]
目标受众：[目标用户]
更新频率：每日 1 篇

风格要求：
- 开头吸引人（痛点/悬念）
- 内容有干货
- 结尾引导互动
```

---

## ⚙️ 工作流配置

```yaml
# xiaohongshu-pipeline.yaml
workflow:
  - name: topic_research
    schedule: "0 8 * * *"  # 每天早上8点
    action: fetch_trending_topics
  
  - name: content_generate
    after: topic_research
    action: generate_caption
  
  - name: image_create
    after: content_generate
    action: generate_cover
  
  - name: schedule_post
    after: image_create
    action: schedule_to_post
    time: "20:00"  # 晚8点发布
```

---

## 📊 数据追踪指标

| 指标 | 目标 |
|------|------|
| 曝光量 | >1000/篇 |
| 点赞率 | >5% |
| 收藏率 | >3% |
| 评论数 | >20/篇 |

---

## 💡 爆款技巧

1. **标题公式**：数字+关键词+悬念
2. **首图**：高清、对比、醒目
3. **内容**：干货+情感共鸣
4. **标签**：热门标签 + 垂直标签
