# 多智能体协作系统

**分类:** 中国特色用例  
**难度:** ⭐⭐

---

## 🎯 痛点

- 复杂任务需要多领域专业知识，单一AI难以胜任
- 多任务协作缺乏系统性分工机制
- 任务进度和状态难以追踪
- 缺乏稳定的迭代和更新能力

---

## ⚡ 核心功能

1. **任务分发** - 根据能力匹配分配任务给子智能体
2. **协同工作** - 多智能体并行/串行协作
3. **进度追踪** - 实时追踪任务状态和进度
4. **结果汇总** - 整合各智能体输出形成最终结果
5. **迭代优化** - 支持多轮迭代持续改进

---

## 🛠️ 所需技能

| 技能 | 用途 |
|------|------|
| `subagents` | 创建和管理子智能体 |
| `message` | 跨智能体消息传递 |
| `feishu_bitable` | 任务状态存储 |
| `feishu_calendar` | 协调各智能体工作时间 |

---

## 📝 Prompt 模板

```
你是一个多智能体协作系统的协调者。

任务场景：[描述需要完成的复杂任务]

协作分工：
1. 子智能体A - [负责领域1]
2. 子智能体B - [负责领域2]
3. 子智能体C - [负责领域3]

请协调各子智能体完成协作，并汇总最终结果。
```

## ⏰ Cron 配置示例

```json

  "schedule": "0 */4 * * *",

  "description": "每4小时同步各智能体状态"
```

## 💡 扩展用法

- **评审机制** - 引入评审智能体把关输出质量
- **知识共享** - 建立智能体间知识库
- **动态调整** - 根据任务难度动态调整分工
