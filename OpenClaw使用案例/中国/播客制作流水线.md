# 播客制作流水线

**分类:** 中国特色用例  
**难度:** ⭐⭐

---

## 🎯 痛点

- 播客内容需要同时发布到小宇宙、喜马拉雅、B站等多个平台
- 手动上传和适配各平台格式耗时
- 素材管理混乱，音频、文案、封面难以统一
- 跨平台数据统计困难

---

## ⚡ 核心功能

1. **多平台发布** - 一键发布到小宇宙、喜马拉雅、B站
2. **内容适配** - 自动转换格式适配各平台要求
3. **素材管理** - 统一管理音频、文案、封面素材
4. **数据汇总** - 跨平台播放数据自动统计
5. **定时发布** - 支持预约发布和时间线安排

---

## 🛠️ 所需技能

| 技能 | 用途 |
|------|------|
| `browser` | 自动化操作各播客平台后台 |
| `web_fetch` | 获取平台API或数据页面 |
| `feishu_bitable` | 素材和数据存储 |

---

## 📝 Prompt 模板

```
你是一个播客制作助手，帮助我管理播客内容的多平台发布。

核心能力：
1. 将音频和文案发布到小宇宙、喜马拉雅、B站
2. 管理播客素材（音频、文案、封面）
3. 汇总各平台的播放数据
4. 定时发布和内容规划

请根据我的需求执行相应操作。
```

---

## ⏰ Cron 配置示例

```json
{
  "schedule": "0 10 * * 6",
  "task": "podcast_analytics",
  "description": "每周六汇总播客数据"
}
```

---

## 💡 扩展用法

- **内容复用** - 将播客内容拆分为短视频分发
- **粉丝互动** - 自动回复各平台评论
- **选题规划** - 根据数据分析优化内容方向
