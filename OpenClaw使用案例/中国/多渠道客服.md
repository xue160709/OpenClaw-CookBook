# 多渠道客服

**分类:** 中国特色用例  
**难度:** ⭐⭐⭐

---

## 🎯 痛点

- 客户分布在微信、抖音、小红书等多平台
- 手动切换平台回复效率低
- 客服响应速度影响转化率
- 难以统一管理客户画像

---

## ⚡ 核心功能

1. **多平台接入** - 抖音 / 小红书 / 微信 / 企业微信
2. **统一收件箱** - 所有渠道消息汇集一处
3. **智能回复** - FAQ 自动回答、问题分类
4. **客户画像** - 汇总历史交互记录
5. **工单流转** - 复杂问题转人工、跟踪处理

---

## 🛠️ 所需技能

| 技能 | 用途 |
|------|------|
| `feishu_im` | 飞书消息 |
| `wechat` | 个人微信 |
| `wechat_work` | 企业微信 |
| `browser` | 抖音/小红书自动化 |
| `memory` | 客户画像 |

---

## 📝 Prompt 模板

```
你是多渠道客服助手，负责统一响应客户咨询。

响应规则：
1. 优先匹配 FAQ，知识库中有答案的直接回复
2. 无法回答的问题转人工，并标记优先级
3. 整理客户基本信息到画像库

FAQ 格式：
问题 | 答案 | 适用渠道

请用专业、友好的语气回复。
```

---

## ⚙️ 渠道配置

```yaml
# multi-channel-customer-service.yaml
channels:
  - name: douyin
    type: browser
    automation: true
    
  - name: xiaohongshu
    type: browser
    automation: true
    
  - name: wechat_work
    type: wechat_work
    corp_id: xxx
    
  - name: feishu
    type: feishu_im

unified_inbox:
  enabled: true
  channel: feishu_bot
```

## 💡 进阶功能

- **情感分析**：识别客户情绪，优先处理负面
