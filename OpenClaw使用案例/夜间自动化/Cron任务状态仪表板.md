# Cron 任务状态仪表板

**分类:** 夜间自动化  
**编号:** #35

## 🎯 痛点
- 部署了大量定时任务（Cron Jobs），难以统一监控
- 任务执行失败无法及时发现
- 缺乏可视化的任务执行状态仪表板

## ⚡ 核心功能
1. 汇总所有Cron任务的配置信息
2. 记录每次任务执行的开始/结束时间和状态
3. 检测任务执行异常（超时、报错）
4. 生成可视化状态仪表板
5. 任务失败时发送告警通知

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| 系统监控 | 读取crontab和系统日志 |
| 可视化 | 生成状态仪表板 |

## 📝 Prompt 模板
```
你是Cron任务监控助手。请汇总所有定时任务的状态并生成仪表板。

任务步骤：
1. 获取当前用户的 crontab 配置：
   ```bash
   crontab -l
   ```
2. 读取系统日志获取任务执行历史：
   ```bash
   grep -r "CRON" /var/log/syslog | tail -n 100
   # 或
   journalctl -u cron --since "24 hours ago"
   ```
3. 分析每个任务的执行情况：
   - 成功次数
   - 失败次数
   - 平均执行时间
   - 上次执行时间
4. 检测异常任务：
   - 连续失败超过3次
   - 执行时间超过预期
   - 未按时执行
5. 生成状态仪表板（Markdown格式）：

```markdown
# Cron 任务状态仪表板

## 📊 概览
- 总任务数: X
- 今日执行: X
- 成功: X | 失败: X

## ✅ 正常任务
| 任务 | 频率 | 上次执行 | 状态 |
|------|------|----------|------|

## 🔴 异常任务
| 任务 | 问题 | 建议 |
|------|------|------|
```

发送报告到指定通知渠道。
```
