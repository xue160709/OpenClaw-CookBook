# 5AM 基础设施健康检查

**分类:** 夜间自动化  
**编号:** #13

## 🎯 痛点
- 夜间服务器可能出现故障，但无人及时发现
- 早上上班时才发现问题，错过了最佳处理时机
- 手动巡检耗时且容易遗漏关键指标

## ⚡ 核心功能
1. 自动检测服务器存活状态（ping、SSH连接）
2. 检查关键服务运行状态（nginx、docker、数据库等）
3. 监控系统资源使用率（CPU、内存、磁盘、IO）
4. 检查日志文件中的错误和异常
5. 检测SSL证书到期时间
6. 汇总健康报告并发送通知

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| healthcheck | 执行基础设施健康检查 |
| 监控 | 收集系统指标和日志 |
| 通知 | 发送告警通知 |

## 📝 Prompt 模板
```
你是基础设施健康检查助手。请在凌晨5点执行服务器全面的健康检查。

检查项目：
1. 服务器存活检测 - ping每台服务器
2. SSH连接测试 - 验证远程访问是否正常
3. 服务状态检查 - nginx、docker、postgresql/mysql、redis等
4. 资源使用率 - CPU、内存、磁盘空间（超过80%告警）
5. 错误日志扫描 - /var/log下的error和critical日志
6. SSL证书检查 - 使用openssl检查证书到期时间
7. 定时任务检查 - cron服务是否正常运行

生成健康报告，格式：
```
## 服务器健康报告 - [日期]

### 总体状态：🟢 正常 / 🔴 异常

### 详细检查：
- 服务器列表及状态
- 异常项及建议处理措施

### 资源使用排名：
- CPU最高3台
- 内存最高3台
- 磁盘最高3台
```

如果发现异常，发送告警通知到指定渠道。
```
