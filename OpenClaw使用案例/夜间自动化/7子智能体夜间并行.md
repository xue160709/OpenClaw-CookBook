# 7 子智能体夜间并行

**分类:** 夜间自动化  
**编号:** #12

## 🎯 痛点
- 需要同时执行多个独立的后台任务，但串行执行效率低下
- 手动管理多个子Agent的启动、监控和结果汇总非常繁琐
- 难以并行处理复杂任务以充分利用计算资源

## ⚡ 核心功能
1. 一次性启动最多7个子Agent并行执行不同任务
2. 实时监控所有子Agent的执行状态
3. 自动汇总各子Agent的执行结果
4. 失败重试机制：单个子Agent失败不影响其他任务
5. 支持任务依赖关系配置

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| subagents | 管理和调度子Agent并行执行 |
| 任务编排 | 配置任务依赖和执行顺序 |

## 📝 Prompt 模板
```
你是任务编排助手，擅长并行调度多个子Agent完成任务。

请执行以下任务：

1. 接收用户的任务列表（最多7个独立任务）
2. 同时启动所有子Agent，每个子Agent负责一个任务
3. 轮询检查各子Agent的执行状态
4. 等待所有子Agent完成后，汇总结果
5. 如果有失败的子Agent，尝试重新执行（最多2次）

任务示例：
- Agent 1: 抓取某个网页数据
- Agent 2: 处理本地文件
- Agent 3: 发送邮件通知
- Agent 4-7: 其他独立任务...

请以JSON格式返回最终汇总结果。
```
