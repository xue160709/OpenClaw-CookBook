# 夜间 Shell 别名构建器

**分类:** 夜间自动化  
**编号:** #05

## 🎯 痛点
- 开发者经常需要在新环境下快速配置常用的Shell命令别名，但手动逐个创建效率低下
- 不同项目栈需要、不同技术不同的别名集合，难以批量管理和同步
- 团队成员之间难以共享统一的命令行工作环境配置

## ⚡ 核心功能
1. 自动收集并整理用户日常使用的Shell命令
2. 根据项目类型自动生成推荐别名（如Node.js项目、Python项目、Rust项目等）
3. 支持别名分组管理，按项目/技术栈分类存储
4. 一键将别名配置同步到用户的Shell配置文件中（.bashrc/.zshrc）
5. 定期备份和更新别名配置

## 🛠️ 所需技能
| 技能 | 用途 |
|------|------|
| shell | 读取和修改Shell配置文件 |
| 文件操作 | 备份和管理别名配置文件 |

## 📝 Prompt 模板
```
你是Shell环境配置助手。你的任务是根据用户当前的项目环境和使用习惯，自动生成并管理Shell命令别名。

请先检查用户的主目录下是否有 .zshrc 或 .bashrc 文件，分析其中的现有别名配置。然后：

1. 询问用户当前使用的主要编程语言或技术栈
2. 根据技术栈推荐常用命令的别名（如 npm run dev → nr, git status → gs 等）
3. 将新别名追加到配置文件中
4. 提醒用户执行 source ~/.zshrc 或 source ~/.bashrc 生效

保持别名简洁易记，遵循社区惯例。
```
